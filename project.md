# Planning Poker AI — Техническая документация

## Описание проекта

Real-time приложение для Planning Poker — инструмента оценки задач в Agile-командах. Позволяет участникам команды анонимно голосовать карточками с числами Фибоначчи, после чего результаты синхронно отображаются всем участникам.

## Механика работы

### 1. Подключение к комнате
- Пользователь вводит ID комнаты или создаёт новую
- Фронтенд устанавливает WebSocket-соединение с сервером
- Сервер создаёт или находит существующую комнату в памяти (Map)
- Пользователь получает уникальный `playerId`

### 2. Голосование
- Каждый участник выбирает карту из колоды
- Голос отправляется на сервер через WebSocket
- Сервер обновляет состояние комнаты и рассылает обновления всем участникам
- Карты других игроков скрыты до момента раскрытия

### 3. Раскрытие результатов
- Хост комнаты (первый подключившийся) нажимает «Показать карты»
- Сервер устанавливает флаг `votesRevealed = true`
- Все участники видят карты друг друга
- Отображается статистика: среднее, минимум, максимум

### 4. Новый раунд
- Хост запускает новый раунд
- Сбрасываются все голоса
- Счётчик раундов увеличивается
- Голосование начинается заново

### Протокол сообщений

**Клиент → Сервер:**
```typescript
type ClientMessage =
  | { type: 'JOIN_ROOM'; roomId: string; playerName: string }
  | { type: 'LEAVE_ROOM'; roomId: string; playerId: string }
  | { type: 'CAST_VOTE'; roomId: string; playerId: string; vote: CardValue }
  | { type: 'REVEAL_VOTES'; roomId: string; playerId: string }
  | { type: 'NEW_ROUND'; roomId: string; playerId: string }
  | { type: 'KICK_PLAYER'; roomId: string; playerId: string; kickedPlayerId: string };
```

**Сервер → Клиент:**
```typescript
type ServerMessage =
  | { type: 'ROOM_STATE'; room: SerializedRoom; playerId: string }
  | { type: 'PLAYER_JOINED'; room: SerializedRoom; player: Player }
  | { type: 'PLAYER_LEFT'; room: SerializedRoom; playerId: string }
  | { type: 'VOTE_CAST'; room: SerializedRoom; playerId: string }
  | { type: 'VOTES_REVEALED'; room: SerializedRoom }
  | { type: 'NEW_ROUND_STARTED'; room: SerializedRoom }
  | { type: 'ERROR'; message: string };
```

## User Stories

### US-1: Создание комнаты
**Как** участник команды,  
**Я хочу** создать новую комнату для Planning Poker,  
**Чтобы** пригласить коллег и начать оценку задач.

**Критерии приёмки:**
- Кнопка «Создать новую комнату» на главной странице
- Автоматическая генерация уникального ID комнаты
- Перенаправление на страницу созданной комнаты
- Возможность скопировать ссылку для приглашения

---

### US-2: Присоединение к комнате
**Как** участник команды,  
**Я хочу** присоединиться к существующей комнате по ссылке или ID,  
**Чтобы** участвовать в голосовании вместе с командой.

**Критерии приёмки:**
- Поле ввода ID комнаты на главной странице
- Валидация введённого ID
- Форма ввода имени перед входом
- Отображение имени пользователя в списке участников

---

### US-3: Голосование картой
**Как** участник команды,  
**Я хочу** выбрать карту для оценки задачи,  
**Чтобы** выразить своё мнение о сложности задачи.

**Критерии приёмки:**
- Отображение полной колоды карт
- Возможность выбрать одну карту
- Визуальное подтверждение выбора
- Блокировка повторного голосования в том же раунде
- Отображение статуса «Проголосовал(а)» для других участников

---

### US-4: Просмотр результатов
**Как** участник команды,  
**Я хочу** увидеть результаты голосования после раскрытия карт,  
**Чтобы** понять расхождение оценок и обсудить их.

**Критерии приёмки:**
- Отображение всех карт участников после раскрытия
- Показ статистики: среднее, минимум, максимум
- Имена участников под каждой картой
- Автоматическое обновление у всех участников одновременно

---

### US-5: Начало нового раунда
**Как** хост комнаты,  
**Я хочу** начать новый раунд голосования,  
**Чтобы** оценить следующую задачу спринта.

**Критерии приёмки:**
- Кнопка «Новый раунд» доступна только хосту
- Сброс всех голосов участников
- Увеличение счётчика раундов
- Уведомление всех участников о начале нового раунда

---

### US-6: Отслеживание участников
**Как** участник команды,  
**Я хочу** видеть список всех участников комнаты,  
**Чтобы** знать, кто присутствует и кто уже проголосовал.

**Критерии приёмки:**
- Список участников с именами
- Индикация статуса голосования (проголосовал/нет)
- Подсветка текущего пользователя
- Отображение иконки 👑 для хоста комнаты
- Счётчик общего количества участников

---

### US-7: Переключение темы
**Как** пользователь,  
**Я хочу** переключаться между светлой и тёмной темой,  
**Чтобы** комфортно работать в любое время суток.

**Критерии приёмки:**
- Кнопка переключения темы в правом верхнем углу
- Мгновенное применение темы ко всем элементам
- Сохранение предпочтений (localStorage)
- Корректное отображение всех цветов в обеих темах

---

### US-8: Копирование ссылки
**Как** участник команды,  
**Я хочу** скопировать ссылку на комнату,  
**Чтобы** быстро пригласить коллег.

**Критерии приёмки:**
- Кнопка «Копировать ссылку» в интерфейсе комнаты
- Копирование полного URL в буфер обмена
- Визуальное подтверждение успешного копирования

---

### US-9: Выход из комнаты
**Как** участник команды,  
**Я хочу** покинуть комнату,  
**Чтобы** завершить участие в сессии.

**Критерии приёмки:**
- Кнопка «Выйти» в интерфейсе комнаты
- Уведомление остальных участников о выходе
- Возврат на главную страницу
- Удаление комнаты при выходе последнего участника

## Запуск проекта

```bash
# Установка зависимостей
npm install

# Запуск в режиме разработки (сервер + фронтенд)
npm run dev

# Сборка для продакшена
npm run build

# Запуск продакшен-версии
npm start
```

**Порты:**
- WebSocket-сервер: `8080` (настраивается через `WS_PORT`)
- Next.js: `3000` (по умолчанию)

## Структура данных

### Room
```typescript
interface Room {
  id: string;              // Уникальный ID комнаты
  name: string;            // Отображаемое имя
  players: Map<string, Player>;  // Участники
  deck: CardValue[];       // Колода карт
  round: number;           // Текущий раунд
  votesRevealed: boolean;  // Флаг раскрытия голосов
  createdAt: number;       // Время создания (timestamp)
}
```

### Player
```typescript
interface Player {
  id: string;              // Уникальный ID игрока
  name: string;            // Имя игрока
  vote: CardValue | null;  // Текущий голос
  isReady: boolean;        // Готовность к игре
  ws?: WebSocket;          // WebSocket-соединение
}
```

## Технологические ограничения

- **Хранение данных**: Комнаты хранятся только в оперативной памяти сервера
- **Персистентность**: При перезапуске сервера все комнаты удаляются
- **Масштабирование**: Отсутствие поддержки нескольких инстансов сервера
- **Безопасность**: Нет аутентификации и авторизации пользователей
